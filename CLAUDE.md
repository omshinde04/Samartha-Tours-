## General Guidelines

### Language & Communication
- All communication must be conducted in **English**.
- Always provide clear, concise, and professional responses.
- For technical issues, include relevant code snippets and error messages.
- For design suggestions, provide visual context and rationale.

### Coding Standards
- **Code Structure**: Follow the component-based architecture as outlined in the project plan.
- **File Naming**: Use PascalCase for React components (e.g., `Header.jsx`) and camelCase for other files (e.g., `utils.js`).
- **Styling**: Use modern CSS practices. Ensure responsive design for both mobile and desktop.
- **Optimization**: Optimize for performance and load times. Avoid unnecessary re-renders.

### Design Standards
- **Responsiveness**: All designs must be fully responsive and mobile-first.
- **Accessibility**: Ensure designs follow accessibility guidelines (WCAG 2.1).
- **Visual Consistency**: Maintain consistency in typography, color palette, and spacing throughout the application.

### Quality Assurance
- **Testing**: Test all code changes to ensure they do not introduce regressions.
- **Error Handling**: Implement robust error handling and fallback mechanisms.
- **Documentation**: Keep documentation updated with all changes and decisions.

### Change Management
- **Scope Creep**: Be vigilant about scope creep. Always verify changes against the original project outline.
- **Communication**: If a change request exceeds the original scope, communicate with the Project Architect for approval.
- **Verification**: After implementing changes, verify that the application still meets all requirements.
